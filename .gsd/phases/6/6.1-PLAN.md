# Plan 6.1: Data Logging Engine

<objective>
Enable the application to record the incoming telemetry stream (Timestamp, Freq, Cap, ResFreq, Imp) into memory while an experiment is active, preparing it for CSV export.
Purpose: Scientific data is useless if it cannot be captured and analyzed later.
Output: Dependencies installed and `SerialService` upgraded with a datalogging array limit/buffer.
</objective>

<context>
Load for context:
- package.json
- src/services/SerialService.ts
</context>

  <files>src/services/SerialService.ts</files>

    Add a public getter `getSessionLog()` returning the array.
