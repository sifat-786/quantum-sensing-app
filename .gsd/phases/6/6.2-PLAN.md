# Plan 6.2: CSV Export Integration

<objective>
Wire up the Download button on the `ExperimentToolbar` to extract the `sessionLog` from `SerialService`, convert it to a CSV string format, save it to the device's local file system, and trigger the native Share sheet so the user can email or save the data.
Purpose: Deliver the core scientific tracking requirement.
Output: A new `ExportUtils` file and an updated highly-functional Toolbar component.
</objective>

<context>
Load for context:
- src/components/ExperimentToolbar.tsx
- src/services/SerialService.ts
</context>

<tasks>
<task type="auto">
  <name>Create Export Utility</name>
  <files>src/utils/ExportUtils.ts</files>
  <action>
    Create a new file `ExportUtils.ts`.
    Write a function `exportSessionToCSV(data: TelemetryData[])`.
    Use standard string building to create CSV headers: `Timestamp,Frequency (Hz),Capacitance (pF),Res. Freq (kHz),Impedance (kOhm)\n`.
    Iterate over `data` to append rows.
    Use `FileSystem.documentDirectory` to establish a path `aqc_export_TIMESTAMP.csv`.
    Use `FileSystem.writeAsStringAsync` to save the file.
    Use `Sharing.isAvailableAsync` and `Sharing.shareAsync` to open the mobile share sheet.
  </action>
  <verify>Syntax is valid</verify>
  <done>Export logic is decoupled and testable</done>
</task>

<task type="auto">
  <name>Bind Download Button</name>
  <files>src/components/ExperimentToolbar.tsx</files>
  <action>
    Import `exportSessionToCSV` and `serialService`.
    Create a handler `handleExport` that grabs `serialService.getSessionLog()` and passes it to the utility.
    Bind this handler to the `onPress` of the Download `<TouchableOpacity>`.
  </action>
  <verify>Syntax is valid</verify>
  <done>UI triggers the export flow</done>
</task>
</tasks>
